# Workout Tracker Application

A complete Spring Boot CRUD web application for tracking workouts and exercises. Built with Spring Data JPA (Hibernate ORM), Thymeleaf, MySQL, and Lombok.

## Features

### Exercise Library (CRUD)
- ✅ View list of all exercises
- ✅ Add new exercises
- ✅ Edit existing exercises
- ✅ Delete exercises

### Workout Management (CRUD)
- ✅ Create workouts with date and notes
- ✅ Edit workout details
- ✅ Delete workouts
- ✅ Add exercises to workouts with sets, reps, and weight
- ✅ View and manage workout entries

### Workout Reports
- ✅ Filter workouts by date range
- ✅ View detailed workout history
- ✅ See all exercises performed in each workout

## Technology Stack

- **Spring Boot 3.5.7**
- **Spring Data JPA** (Hibernate ORM)
- **Thymeleaf** (Template Engine)
- **MySQL** (Database)
- **Lombok** (Reduces Boilerplate)
- **Bootstrap 5** (UI Framework)

## Architecture

### Entities
- **Exercise**: Represents an exercise with name and muscle group
- **Workout**: Represents a workout session with date and notes
- **WorkoutEntry**: Links exercises to workouts with sets, reps, and weight

### Repositories (ORM Only)
- `ExerciseRepository` - Extends `JpaRepository<Exercise, Long>`
- `WorkoutRepository` - Extends `JpaRepository<Workout, Long>` with custom method `findByWorkoutDateBetween`
- `WorkoutEntryRepository` - Extends `JpaRepository<WorkoutEntry, Long>`

### Controllers
- `ExerciseController` - Handles all exercise-related operations
- `WorkoutController` - Handles workout CRUD and entry management
- `HomeController` - Redirects root URL to exercises page

## Database Schema

The database schema is automatically generated by Hibernate from JPA entities:

- **exercises** table (exercise_id, name, muscle_group)
- **workouts** table (workout_id, workout_date, notes)
- **workout_entries** table (entry_id, sets, reps, weight, workout_id, exercise_id)

## Prerequisites

- Java 17 or higher
- Maven 3.6+
- MySQL 8.0+ (or compatible)

## Setup Instructions

### 1. Database Setup

Make sure MySQL is running and create a database (or it will be auto-created):

```sql
CREATE DATABASE IF NOT EXISTS workoutdb;
```

### 2. Configuration

Update `src/main/resources/application.properties` if needed:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/workoutdb?createDatabaseIfNotExist=true&useSSL=false&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=your_password_here

spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

**Note**: If your MySQL password is not empty, update the `spring.datasource.password` property.

### 3. Build the Project

```bash
./mvnw clean install
```

Or on Windows:
```bash
mvnw.cmd clean install
```

### 4. Run the Application

```bash
./mvnw spring-boot:run
```

Or on Windows:
```bash
mvnw.cmd spring-boot:run
```

### 5. Access the Application

Open your browser and navigate to:
```
http://localhost:8080
```

The application will redirect to the Exercise Library page.

## Usage

### Managing Exercises

1. Click **"Add New Exercise"** to create an exercise
2. Enter the exercise name and muscle group
3. Use **Edit** to modify or **Delete** to remove exercises

### Creating Workouts

1. Navigate to **Workouts** from the navigation
2. Click **"Create New Workout"**
3. Enter the workout date and optional notes
4. Save the workout
5. After saving, add exercises using the dropdown:
   - Select an exercise
   - Enter sets, reps, and weight
   - Click **Add** to add the entry to the workout

### Generating Reports

1. Click **"Reports"** from the navigation
2. Select a **From Date** and **To Date**
3. Click **"Generate Report"** to view workouts in that date range

## Key Features

✅ **Pure ORM Approach**: All database operations use Spring Data JPA repository methods  
✅ **Auto-generated Schema**: Database tables are created automatically from JPA entities  
✅ **No Raw SQL**: Zero SQL queries - everything uses ORM  
✅ **Cascade Operations**: Deleting a workout automatically deletes its entries  
✅ **Date Range Filtering**: Uses repository method naming conventions (`findByWorkoutDateBetween`)

## Troubleshooting

### Connection Issues
- Verify MySQL is running: `mysql -u root -p`
- Check database credentials in `application.properties`
- Ensure the database `workoutdb` exists or can be created

### Port Already in Use
- Change the port in `application.properties`: `server.port=8081`

### Schema Not Created
- Check that `spring.jpa.hibernate.ddl-auto=update` is set
- Verify database connection is successful
- Check console logs for Hibernate SQL output
